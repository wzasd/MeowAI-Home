# Phase 4 WebUI: Thread 管理增强实现日记

**日期**: 2026-04-11

## 已完成工作

### Thread 操作功能

给 ThreadItem 添加了完整的管理功能：

| 功能 | 交互方式 | 状态显示 |
|------|----------|----------|
| 重命名 | 更多菜单 → 重命名 | 内联编辑框 |
| 归档 | 更多菜单 → 归档 | 删除线 + 「已归档」标签 |
| 删除 | 更多菜单 → 删除 | 确认对话框 |

### 代码变更

**ThreadItem.tsx** (完全重写)
- 添加操作菜单（MoreVertical 图标，悬停显示）
- 内联编辑模式（input 自动聚焦、Enter 保存、Escape 取消）
- 归档状态显示（删除线 + 标签）
- 加载状态（防止重复操作）
- 点击外部自动关闭菜单

**threadStore.ts**
- 添加 `renameThread(id, name)` - 乐观更新本地状态
- 添加 `archiveThread(id)` - 切换归档状态
- 现有 `deleteThread(id)` - 已存在

**ThreadSidebar.tsx**
- 从 store 获取新方法
- 传递给 ThreadItem 组件

**api/client.ts**
- `api.threads.rename(id, name)`
- `api.threads.archive(id)`

### UX 细节

1. **操作菜单**
   - 悬停在 ThreadItem 上显示「更多」按钮
   - 点击外部或选择操作后关闭

2. **重命名**
   - 自动聚焦并选中全文
   - Enter = 保存，Escape = 取消
   - 失焦自动保存

3. **归档**
   - 归档后显示删除线和「已归档」标签
   - 再次归档 = 取消归档

4. **删除**
   - confirm() 确认对话框
   - 显示对话名称以防误删
