## Phase 4.2 任务分解

### 任务 1: 让技能"活"起来

**阿橘的理解**：
"现在技能装在 `~/.meowai/skills/` 里，但是对话时还不会自动用。需要在 A2AController 里加个路由器，用户一说'写代码'，就给我加载 TDD 技能。"

**墨点的分析**：
"……集成点在 A2AController.execute()。流程是：
1. 用户发送消息
2. ManifestRouter 匹配触发词
3. SkillLoader 加载技能
4. 注入技能上下文到系统提示
5. 调用 LLM"

**花花补充**：
"还要在 CLI 显示提示，告诉用户技能已激活。就像这样：
```
💡 技能已激活: TDD 工作流
```

### 任务 2: 扩展到 25 个技能

**铲屎官的计划**：
"我们还需要 19 个技能：

**高级功能 (6个)**:
- self-evolution - 自我进化
- cross-thread-sync - 跨 Thread 协同
- deep-research - 深度调研
- schedule-tasks - 定时任务
- writing-skills - 编写技能
- incident-response - 事故应急

**MCP 集成 (3个)**:
- pencil-design - Pencil 设计
- rich-messaging - 富媒体消息
- browser-automation - 浏览器自动化

**用户体验 (3个)**:
- workspace-navigator - Workspace 导航
- browser-preview - 浏览器预览
- image-generation - AI 图片生成

**健康与训练营 (2个)**:
- hyperfocus-brake - 健康提醒
- bootcamp-guide - 训练营引导

**协作流程补充 (2个)**:
- collaborative-thinking - 协作思考
- feat-lifecycle - 功能生命周期

**开发流程补充 (3个)**:
- writing-plans - 编写计划
- worktree - Worktree 隔离
"

**阿橘**（数了数）： "已经有 6 个了，再加 19 个...哇，25 个技能！这样技能集就完整了！"

### 任务 3: 技能链提示

**墨点的建议**：
"……技能有 `next` 字段，指向下一个技能。比如 TDD → quality-gate。我们需要在技能上下文里加个提示：
```
**建议下一步**: 使用 Quality Gate 技能
```
这样用户就知道接下来该做什么。"

**花花**： "对！这样技能链就形成了完整的工作流引导。"
